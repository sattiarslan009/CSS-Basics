Cascading Style Sheet

Cascading Style Sheets is a style sheet language used for describing the presentation of a document written in a markup language such as HTML or XML.
CSS is a cornerstone technology of the World Wide Web, alongside HTML and JavaScript

Cascading means in CSS

In CSS, "Cascading" refers to the way styles are determined and applied to HTML elements when multiple conflicting styles are defined.
The term "Cascading" is an important aspect of the CSS (Cascading Style Sheets) language, and it describes the order of priority that the browser uses to apply styles.
The CSS styles can come from different sources, and they have different levels of specificity and importance. These sources, in order of increasing priority, are:

  • User Agent Stylesheet:

    This is the default styling applied by the browser itself. Every HTML element has some default styles provided by the browser.

  • External Stylesheets:

    These are external CSS files that you link to your HTML document. Styles defined here can be applied globally to your entire website.

  • Internal Stylesheets:

    These are styles defined within the "style" element in the HTML document "head" They affect the specific HTML document they are included in.

  • Inline Styles:

    These are styles defined directly on individual HTML elements using the style attribute. They have the highest specificity and will override external and internal styles.

  • Specificity and Importance:

    If multiple rules with the same selector target an element, CSS uses specificity to determine which rule should be applied.
    Additionally, you can use the !important declaration to give a style rule the highest priority.

  • Order of Appearance:

    If two rules have the same specificity and importance, the one that appears later in the stylesheet or HTML document takes precedence.

Why we use CSS

CSS (Cascading Style Sheets) is used in web development for several important reasons

  • Separation of Concerns:

    CSS allows you to separate the content (HTML) of a web page from its presentation (styling).
    This separation makes it easier to manage and maintain web content.
    Developers can focus on the structure and semantics of the HTML while designers work on the visual presentation using CSS.

  • Consistency:

    CSS enables you to create a consistent look and feel across a website or web application.
    By defining styles centrally, you can ensure that headings, paragraphs, buttons, and other elements have a uniform appearance throughout the site.

  • Flexibility:

    With CSS, you have fine-grained control over the layout and styling of web elements.
    You can change the appearance of an entire site by modifying a single CSS file, making it highly adaptable to design changes or user preferences.

  • Accessibility:

    CSS allows you to improve the accessibility of your web content. You can use CSS to control the order in which content is presented (e.g., using flexbox or grid layout)
    It make text more readable, and ensure that web pages are responsive to different devices and screen sizes.

  • Load Time and Bandwidth:

    Separating styles from content reduces the size of HTML files, making web pages load faster.
    This can improve the user experience, especially on slower internet connections and mobile devices.

  • Maintenance:

    When you want to make global design changes, such as adjusting colors or fonts, you can do so by editing a single CSS file.
    This is much more efficient than manually updating the style of each HTML element individually

  • Cross-Browser Compatibility:

    CSS helps ensure that your website looks consistent and works correctly across different web browsers.
    Browsers may have their own default styles, and CSS allows you to override these defaults and achieve a consistent appearance.

  • Animations and Interactivity:

    CSS includes support for animations and transitions, allowing you to create interactive and visually engaging user interfaces without relying on JavaScript.

  • Print Styling:

    CSS can be used to create print stylesheets, which define how a web page should appear when printed.
    This enables you to optimize the printed output of web content.

  • Third-Party Integration:

    When using CSS frameworks or libraries (e.g., Bootstrap, Materialize)
    you can leverage pre-designed styles and components to expedite web development and achieve a consistent design language.

Stylesheet in CSS

Stylesheets allow web developers and designers to control the layout and appearance of web content, making it visually appealing, consistent, and user-friendly.
They help maintain a clear separation between content (HTML) and presentation (CSS), facilitating easier maintenance and updates of web pages.
CSS (Cascading Style Sheets), style sheets are documents or files that contain a set of rules and instructions used to define the visual presentation and formatting of HTML (or XML) documents.
These rules specify how HTML elements should be displayed on a web page, including aspects such as layout, colors, fonts, spacing, and other stylistic properties.

Ways of adding CSS to HTML

Three are three main ways of adding CSS to HTML

  • Inline Style:

    Inline styles are applied directly to individual HTML elements using the style attribute.
    These styles have the highest specificity and apply only to the element they are associated with.

  • Internal Style:

    An internal stylesheet, also known as an embedded stylesheet, is defined within the "style" element in the "head" section of an HTML document.
    Styles defined here apply only to that specific HTML document. This is useful when you need styles unique to a single page.

  • External Style:

    An external stylesheet is a separate CSS file with a .css extension that is linked to an HTML document using the "link" element within the HTML document's "head" section.
    This allows you to define styles in one central location and apply those styles to multiple HTML documents.
    External stylesheets are widely used for maintaining a consistent design across a website.

Limitations of HTML in Styling Before CSS:

Before the introduction of CSS, HTML was primarily responsible for both content and presentation. This had several limitations.

  • Limited Styling Options:

    HTML's styling capabilities were very limited. You could change basic attributes like text color, font size, and background color
    But complex layout and design were challenging to achieve.

  • Inconsistent Design:

    Without a standardized way to apply styles, websites often had inconsistent designs.
    Styling had to be repeated for each HTML element, leading to code redundancy.

  • Maintenance Challenges:

    Any design changes required manual updates to every HTML page.
    This was time-consuming and error-prone, especially for larger websites.

  • Lack of Responsiveness:

    HTML alone couldn't provide responsive design.
    Websites looked the same regardless of the user's screen size, making them less user-friendly on different devices.

  • Accessibility Issues:

    HTML couldn't address accessibility concerns effectively.
    Ensuring a good experience for users with disabilities was difficult without proper styling control.

  • Print Styling Difficulties:

    Creating print-friendly versions of web pages was challenging, often resulting in poor-quality printed output.
